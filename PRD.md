## 一、产品概述

一个**轻量、可自托管**的多平台热搜聚合网站。

首页以卡片网格形式集中展示微博、知乎、B 站等平台热榜，用户**无需登录**即可快速浏览当前热点。

**产品定位**：信息聚合工具 + 全栈学习项目。

---

## 二、目标用户

| 用户类型 | 核心诉求 |
|---------|---------|
| 上班族、学生 | 快速扫一眼各平台热点，节省切换时间 |
| 开发者 / Vibe Coding 学习者 | 理解全栈项目结构、爬虫策略、缓存设计 |

## 三、产品范围

### 3.1 MVP（必须实现）

#### 1. 首页卡片网格

- **平台数量**：≥ 3 个
  - 微博热搜
  - 知乎热榜
  - B 站热门

- **每卡片内容**：
  - 平台名称 + 图标（可选）
  - 榜单名（如"微博热搜榜"）
  - 榜单条目：Top 10～20
    - 排名（数字）
    - 标题
    - 跳转原文链接（新标签页打开）

#### 2. 数据接口（自建后端）

**按平台获取**：

```
GET /api/hot/:source
source ∈ { weibo, zhihu, bilibili }
```

**可选聚合接口**：

```
GET /api/hot（返回全部平台）
```

统一 JSON 格式（见技术设计部分）。

#### 3. 缓存机制

- 每个平台独立缓存
- TTL：5～10 分钟（**默认 5 分钟**）
- 页面展示："**更新于 x 分钟前**"

#### 4. 异常与降级

- ✅ 单个平台抓取失败 → **不影响其他卡片**
- ✅ 失败卡片展示明确提示："**暂时无法获取，请稍后重试**"
- ❌ **禁止**整页崩溃或白屏

#### 5. 部署要求

- **代码托管**：GitHub（公开）
- **公网访问**：HTTPS
- **部署平台**：不限（推荐 Vercel + Railway）

### 3.2 后续迭代（可选）

## 四、界面与交互要求

### 4.1 设计原则

- **清晰 > 花哨**
- 信息密度适中，参考「今日头条」与「今日热榜」

### 4.2 响应式布局

| 设备 | 卡片列数 |
|------|:--------:|
| 桌面（≥ 1024px） | 2～3 列 |
| 平板（768px - 1023px） | 1～2 列 |
| 手机（< 768px） | 单列 |

### 4.3 页脚必含内容

- 学习项目说明
- 数据来源声明
- 非商用声明
- 最后更新时间

## 五、技术栈（建议但不强制）

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React + TypeScript + Vite | 快速、类型安全 |
| 样式 | CSS / Tailwind | 响应式优先 |
| 后端 | Node.js + Express | 轻量中转 |
| 部署 | Vercel（前端）+ Railway（后端） | 示例组合 |

---

## 六、成功标准（可度量）

| 指标 | 标准 |
|------|------|
| 数据拉取 | ✅ 3 个平台真实数据可正常拉取 |
| 缓存生效 | ✅ 更新间隔 ≤ 10 分钟 |
| 异常容错 | ✅ 单个平台异常时，其它卡片可展示数据 |
| 首屏速度 | ✅ 普通用户打开 URL 后 3 秒内看到内容 |
| 声明合规 | ✅ 页脚声明清晰，无隐晦商业化内容 |

---

## 七、风险与约束

| 类型 | 说明 |
|------|------|
| **上游稳定性** | 第三方热榜接口可能变更、返回结构变化或限制 IP |
| **反爬措施** | 部分平台需伪装 UA / 添加延迟，必要时降级为 RSS 或非官方 API |
| **使用性质** | 学习项目，页脚必须标注"**非商用 · 数据来自第三方平台**" |
